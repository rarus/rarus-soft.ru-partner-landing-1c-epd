## Скачивание файлов с репозитория
### Через HTTPS
1. Убедитесь, что у вас установлен Git. Если нет, скачайте и установите с официального сайта.
2. Откройте терминал (командную строку).
3. Перейдите в директорию, где хотите разместить проект.
4. Выполните команду 
```
git clone https://github.com/rarus/rarus-soft.ru-partner-landing-1c-epd.git .
```
[^1].
5. Введите свои учетные данные GitHub, если будет запрошено.
[^1]Обратите внимание, что папка для выполнения команды должна быть пустая.
### Через SSH
1. Убедитесь, что у вас установлен Git и настроены SSH-ключи для GitHub.
2. Если SSH-ключи не настроены, следуйте инструкции GitHub.
3. Откройте терминал (командную строку).
4. Перейдите в директорию, где хотите разместить проект.
5. Выполните команду git@github.com:rarus/rarus-soft.ru-partner-landing-1c-epd.git.
### Через архив (GZ)
1. Перейдите на страницу репозитория на GitHub.
2. Нажмите на кнопку "Code" и выберите "Download ZIP".
3. После загрузки распакуйте архив в нужную директорию.
4. Альтернативно, если вам нужен именно GZ-архив:
   * Перейдите на страницу релизов репозитория.
   * Скачайте архив source code (tar.gz).
   * Распакуйте архив командой: 
```bash 
tar -xzvf *название архива*
```
   
## Развертывание на хостинге
### Общие шаги для любого хостинга
1. Подготовьте файлы проекта (скачайте с репозитория одним из способов выше). 
2. Убедитесь, что у вас есть доступ к хостингу (FTP/SFTP данные или SSH-доступ).
### Через FTP/SFTP
1. Подключитесь к хостингу с помощью FTP-клиента (FileZilla, WinSCP и т.д.). 
2. Перейдите в директорию, в которой будет размещен лендинг. Это может быть как корневая директория (обычно это public_html, www, htdocs или аналогичная), так и вложенная.
3. Загрузите все файлы проекта в эту директорию. 
4. Убедитесь, что файл index.html находится в корне директории.
### Через SSH
1. Подключитесь к серверу по SSH. 
2. Перейдите в директорию, в которой будет размещен лендинг:
```bash 
cd *путь в файловой системе*
```
3. Если вы скачали архив, загрузите его на сервер и распакуйте:
```bash 
scp *путь в файловой системе*/*название архива*.tar.gz username@server:/*путь в файловой системе*/
tar -xzvf *название архива*.tar.gz
```
4. Если вы используете Git, вы можете клонировать репозиторий напрямую на сервер: 
```
git@github.com:rarus/rarus-soft.ru-partner-landing-1c-epd.git . 
```
(обратите внимание на точку в конце, она означает клонирование в текущую директорию, она должна быть пустой)

## Файловая структура проекта

```
├──assets
    ├──bootstrap
    ├──css
    ├──fonts
        ├──Manrope
    ├──images
    ├──js
├──index.html
├──README.md
```

* Папка `assets` — основная папка с ресурсами проекта
    * Папка `bootstrap` — файлы библиотеки Bootstrap
    * Папка `css` — стили лендинга, в которой хранится основной файл style.css со всеми стилями
    * Папка `fonts` — шрифты
        * Папка `Manrope` — основные шрифты проекта
        * Файл `fonts.css` — стили для шрифтов: начертания, пути к файлам и т.д.
    * Папка `images` — все изображения для лендинга
    * Папка `js` — JavaScript-файлы проекта
* Файл `index.html` — основной файл лендинга
* Файл `README.md` — документация по установке и работе с проектом

## Персонализация данных лендинга
### Шапка сайта
1. Заменить лого сайта (формат svg, размер 120*50 пикселей).
### Подвал сайта
1. Заменить лого сайта (формат svg, размер 120*50 пикселей).
2. Заменить контактные данные: телефон, email, адрес. Например:
```html
<ul class="footer__list list">
    <li class="footer__item">
        <a class="footer__link link link--black" href="tel:+74950000000" title="Позвонить">
            +7 (495) 000-00-00
        </a>
    </li>
    <li class="footer__item">
        <a class="footer__link link link--black" href="mailto:epd@example.ru" title="Написать">
            epd@example.ru
        </a>
    </li>
    <li class="footer__item">
        Москва, ул. Красная, 56
    </li>
</ul>
```
3. Указать актуальную ссылку на страницу с политикой конфиденциальности
```html
   <a class="footer__doc-link footer__link link" href="" title="Политика конфиденциальности" target="_blank">
   Политика конфиденциальности
   </a>
```
### Формы «Заказать звонок», «Подключить», «Подобрать пакет», «Попробовать 1С-ЭПД», «Получить консультацию»
1. В установленном шаблоне лендинга логика отправки и сохранения данных с форм не реализована. Этот функционал необходимо настроить вашему веб-разработчику по аналогии с остальными формами на сайте.
2. При необходимости можно раскомментировать верстку сообщений об успешной или неуспешной отправке заявки и добавить логику их вывода.
3. Указать актуальную ссылку на страницу с политикой конфиденциальности.
### Блок «Остались вопросы?»
1. Заменить контактные данные: телефон, email, адрес. Например:
```html
 <li class="contact__contacts-item">
    <img src="assets/images/phone.svg" class="contact__icon icon" width="40">
    <div class="contact__info">
        <p class="contact__type text-uppercase">Телефон</p>

    </div>
</li>
<li class="contact__contacts-item">
    <img src="assets/images/mail.svg" class="contact__icon icon" width="40">
    <div class="contact__info">
        <p class="contact__type text-uppercase">E-mail</p>
        <a href="mailto:epd@example.ru" class="contact__link link link--black">epd@example.ru</a>

<li class="contact__contacts-item">
```
